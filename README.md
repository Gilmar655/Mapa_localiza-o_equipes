# Site corrigido para GitHub Pages

Arquivos prontos para publicar na raiz do repositório.

## Importante
Este pacote usa um `index.html` autossuficiente: o CSS, o JavaScript, a logomarca Enel Brasil, a foto real e os dados dos projetos estão incorporados no próprio arquivo. Isso evita o problema de publicar sem layout, sem imagem ou sem dados no GitHub Pages.

## Como publicar
1. Descompacte o ZIP.
2. Envie o arquivo `index.html` e o arquivo `.nojekyll` para a raiz do repositório.
3. No GitHub, acesse **Settings > Pages**.
4. Em **Build and deployment**, selecione a branch principal e a pasta `/root`.
5. Aguarde a publicação.
